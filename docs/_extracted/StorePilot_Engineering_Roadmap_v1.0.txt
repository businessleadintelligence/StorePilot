Engineering Roadmap — Executive Summary
This document is the complete engineering roadmap for StorePilot v1.0. It breaks development into 10 sequential phases, from environment setup through public launch. Each phase is fully specified with objectives, deliverables, dependencies, engineering tasks, risks, and a definition of done.
All phases and tasks are derived exclusively from the StorePilot Founder Specification v1.0. No features have been invented or added beyond that specification.

Throughout all phases, tasks are labeled with the following priority:

The AI/deterministic boundary is a non-negotiable architectural constraint throughout the entire roadmap. All numerical calculations must be implemented in deterministic engine functions. AI calls receive pre-computed values. Violations of this rule in any phase must be treated as critical bugs.

Appendix: Cross-Phase Engineering Standards
A.1 The AI/Deterministic Boundary (All Phases)
This constraint applies to every phase and every engineering task. AI calls receive pre-computed values from deterministic engines. AI never performs arithmetic. Any violation must be treated as a P1 bug regardless of the phase it is discovered in.
Allowed (AI Layer)
Forbidden (AI Layer)
Generating natural-language explanations
Calculating health scores, revenue impacts, or confidence values
Writing recommendation text
Computing priority scores or urgency values
Producing COO Briefing narrative
Estimating monetary figures not provided in prompt
Interpreting pre-computed trend data
Making statistical comparisons or percentage calculations
Answering merchant questions using provided context
Fabricating numbers when context is missing
A.2 V1 Feature Exclusions (All Phases)
The following items must not be built, stubbed, or scaffolded in any phase of V1 development:
WooCommerce, Magento, BigCommerce, or any non-Shopify platform support
Google Search Console, Meta Ads, Google Ads, Klaviyo, Zendesk, Gorgias integrations
Landing Page AI Analysis, Theme AI Analysis, App Impact Analysis
Autonomous Agents or Auto-Fix Actions (AI writing back to Shopify)
Competitor Intelligence
Multi-Platform Support
A.3 Definition of 'Done' Standards (All Phases)
Every API endpoint has input validation — invalid requests return 400 with descriptive error
Every database write is wrapped in a try/catch — failures are logged with storeId and operation context
All multi-tenant queries are scoped by storeId — no query can return data across store boundaries
All monetary values are in the store's configured currency — no assumed USD
All timestamps are stored in UTC (TIMESTAMPTZ) — displayed in merchant's configured timezone
A.4 Success Criteria Verification (Phase 9)
The product is ready for public launch when merchants can answer all five Founder Specification success criteria without manually analyzing dashboards:
What happened?
Answered by: Business Timeline, Issues Center, Daily COO Briefing
Why did it happen?

What should I do next?

Did previous recommendations work?
Answered by: Recommendation Outcomes, AI COO Impact Score
What changed over time?
Answered by: Business Timeline, Weekly COO Report, Monthly Executive Review
